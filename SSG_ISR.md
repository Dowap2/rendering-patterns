## 🧱 SSG(Static Site Generation) 개념 정리

### 🧠 개념

- 빌드 타임에 HTML을 **미리 생성해 정적 파일로 저장**.
- 배포 시 CDN에 올려두고, 사용자가 접속하면 **서버 연산 없이 즉시 HTML 제공**.

---

### ✅ 장점

| 항목                | 설명                                            |
| ------------------- | ----------------------------------------------- |
| 가장 빠른 응답 속도 | 정적 파일 그대로 전달되기 때문에 서버 계산 없음 |
| 서버 부하 없음      | 요청이 늘어나도 성능 저하 없음                  |
| SEO 유리            | HTML에 콘텐츠가 포함되어 검색엔진에 노출 좋음   |

---

### ❌ 단점

| 항목             | 설명                                               |
| ---------------- | -------------------------------------------------- |
| 페이지 갱신 불편 | 최신 데이터를 반영하려면 **재빌드 + 재배포** 필요  |
| 개인화 어려움    | 사용자별 맞춤 페이지 제공 불가능 (정적 파일이므로) |

---

### 🔍 한 줄 요약

SSG = **가장 빠르고 안정적이지만 최신 데이터 반영이 어렵고 개인화가 불가**

## ⚡ ISR(Incremental Static Regeneration) 개념 정리

### 🧠 개념

- 기본 동작은 **SSG처럼 정적 HTML 제공**.
- 하지만 **설정한 재검증 주기(revalidate 주기)마다 페이지를 다시 생성해 최신 데이터 반영**.

---

### ⚙️ 동작 흐름

1. 첫 요청 → 기존 정적 HTML 반환 (매우 빠름)
2. 백그라운드에서 새로운 HTML을 재생성
3. 주기(revalidate) 도달 후 → **새 HTML로 자동 교체**

---

### ✅ 장점

| 항목          | 설명                                         |
| ------------- | -------------------------------------------- |
| 빠른 속도     | 기본적으로 정적 페이지 제공                  |
| 자동 업데이트 | 수동 재빌드 없이 주기마다 최신 데이터로 갱신 |
| SEO 유리      | HTML 내 콘텐츠 존재                          |

---

### ❌ 단점

| 항목              | 설명                                               |
| ----------------- | -------------------------------------------------- |
| 첫 재생성 시 지연 | 특정 페이지가 처음 갱신될 때 요청자가 느려짐       |
| 실시간은 아님     | 업데이트 주기 사이에는 데이터 변화가 반영되지 않음 |

---

### 🔍 한 줄 요약

ISR = **SSG의 속도 + 자동 갱신**  
하지만 **첫 리프레시 사용자 지연과 실시간성 부족**이 단점
